Kaggle Digit Recognizer 实战

1.KNN

KNN算法其实非常直观，就是看和哪个数字比较像，就判断为哪个数字，类似于我们的死记硬背模式。虽然有点土，但是理论上完全说得过去。

直观而言，KNN算法就是计算训练集中样本与测试集的特征维度空间距离（简称距离），最终选出与测试集距离排行最小的的K个样本，作为判断测试集的标准，是哪个数字就判断为哪个数字。

在测试过程中发现KNN运算耗时非常久，久到离谱（我可以吃个饭再回来），因此对调试参数失去耐心，匆忙结束。

最终得分有0.96左右

2.LR

Logistic Regression就是传说中的万金油算法，主要适用于线性问题，但应付各种题目都有不错的得分。

但是这里显然不太适合用LR，因为这个题目偏于非线性，但其实最后结果也不是很差。

这个算法直观而言，就是评估每个像素点，颜色深一点可能偏向于判断目标数字，或者偏向其他数字。但其实数字对于人而言主要依靠结构识别。我们对于文字的识别大多是根据它们的结构，因此LR容易受到干扰的因素广泛存在。

另外一个点 ，LR训练慢（相对于KNN以及SVM已经比较快了）但是测试快。最终得分能有0.92左右

3.PCA+SVM

先说PCA

主成分分析（Principal Component Analysis，PCA），是一种统计方法。通过正交变换将一组可能存在相关性的变量转换为一组线性不相关的变量，转换后的这组变量叫主成分。

简单直观而言，就是降维，将样本原来的n维空间通过消减维度变为k维空间。

举个梨，全班同学的成绩语文相差很大，从50到90分布，但是英语成绩大家都考到90多分，如果两个成绩都用同样的比重区分排名，那么英语的作用就不这么明显，而且需要考虑两科成绩。
这时如果我新建一个新变量，对两科取不同权重，这样计算出来的新维度即能保留原来的特征，又能达到降维的效果。

然后是SVM

就是支持向量机，起源久远，可以追溯到1956年的感知机算法（感知机算法的目的是通过不断的试错寻找一个合适的超平面hyperplane）

支持向量机算法的学习策略是在训练集中寻找一个超平面，将n维空间分隔开，两类不同的数据分处超平面的正负向，并使间隔最大化。而两类数据距离超平面最近的数据点，就是支持向量。

最终得分0.98
